# Основной файл конфигурации

Основной файл конфигурации сервера Apache находится по пути `*папка apache*/conf/httpd.conf`.

# Дополнительные файлы конфигурации

Если создать в папке файл `.htaccess`, то в нем можно прописывать параметры конкретно к этой папке, которые сервер подгрузит при запуске. Особенность такого файла так-же в том, что сервер подгружает параметры из него при каждом обращении к папке, благодаря чему пропадает необходимость перезапуска сервера при изменении этого файла (как в случае с глобальным файлом конфигурации - при его изменении необходимо перезапустить сервер).

# Директивы

Параметры в файле конфигурации задаются директивами, которые разделяются на однострочные и блочные.

## Однострочные директивы

`SRVROOT "F:/Apps/Apache"` - пример однострочной директивы (она обозначает, грубо говоря, что значение переменной `SRVROOT` равна `"F:/Apps/Apache"`). В однострочные директивы можно передавать несколько параметров - `SRVROOT "F:/Apps/Apache" "something_else"`.

## Блочные директивы

Пример блочной директивы:

```
<Directory />
    AllowOverride none
    Require all denied
</Directory>
```

В данном случае это кусок из главного конфигурационного файла, который задает параметры всем папкам сервера Apache.

# Основные директивы

-   `ServerRoot` - корень сервера
-   `Listen` - порт/ip-адрес для запросов
-   `LoadModule` - подключение модуля
-   `DocumentRoot` - папка с документами (там, где находится `index.html`)
-   `DirectoryIndex` - название главного файла для загрузки (стандартное значение - `index.html`)
-   `Include` - подключение файлов конфигурации

# Директивы директории

-   `Require`
-   `Options`
-   `AllowOverride`
-   `IndexIgnore`
-   `ReadmeName`, `HeaderName`
