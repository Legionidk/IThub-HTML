# Синтаксис тегов
Теги пишутся в `< >`, внутри указывается буква нужного тега, например: `<b>`. Синтаксис **HTML** не чувствителен к регистру (`<b>` то же самое, что и `<B>`). 

Чтобы написать тег ввиде текста внутри другого тега необходимо использовать специальные символы. Ниже приведена краткая таблица символов.
| HTML код |  Описание             | Символ        |
| -------- |  -------------------- | ------------- |
| &nbsp    |  Неразбиваемый пробел |               |
| &quot    |  Прямая кавычка       | "             |
| &amp     |  Амперсанд            | &             |
| &lt      |  Знак "меньше"        | <             |
| &gt      |  Знак "больше"        | >             |
| &copy    |  Копирайт             | ©             |
| &reg     |  Зарегистрировано     | ®             |
| &trade   |  Торговая марка       | TM            |

Больше символов можно найти на [symbl.cc](https://symbl.cc/en/html-entities/).
# Виды тегов
В **HTML** есть два вида тегов:
- Парные
    - `<div> Это пример парного тега </div>`
- Одиночные
    - `<img src="пример_одиночного_тега.png" />`

Главное отличие - **у парных тегов есть закрывающая часть**, а у одиночных нет.
# Атрибуты
У тегов есть **атрибуты**:
- Встроенные атрибуты:
    - `<img src="image.png" />` - атрибут `src` встроенный
- Кастомные атрибуты:
    - `<div data-id=123> Example </div>` - атрибут `data-id` кастомный, то есть создан пользователем для своих нужд

Кастомные атрибуты обычно помечаются `data-` и в дальнейшем могут использоваться в **backend'е**.

# Примеры
Примеры использования тегов можно увидеть в [здесь](index.html). Там не представлены некоторые теги, а именно `<header>`, `<main>` и `<footer>` из-за их простоты и схожести с обычными `<div>` контейнерами:
- `<header>` - то же самое, что и `<div id=header>`
- `<main>` - то же самое, что и `<div id=main>`
- `<footer>` - то же самое, что и `<div id=footer>`

Все они нужны для разделения сайта на составляющие, где `<main>` - главный контент сайта, `<header>` - шапка сайта, `<footer>` - "подвал" сайта. Эти теги были добавлены в **HTML5** и призваны упростить объединение элементов для указанных секций сайта.
